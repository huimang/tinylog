TinyLog is built for high-density log storage and low-memory log browsing. It turns huge plaintext logs into a compact .tog format and keeps large-file navigation fast.

  • Business-facing Java logging API
  • Chunked .tog storage for better density
  • Interactive Rust viewer with search and jump

Why TinyLog

Built around two real log pain points

Storage overhead

Plaintext logs repeat structure and metadata on every line. TinyLog compacts that redundancy into a denser binary layout.

Read amplification

Huge log files are expensive to scan in memory. TinyLog uses trunk-based reads to keep browsing and filtering lightweight.

Workflow fit

Applications can write through a Java SDK, and operators can inspect the result through dedicated Rust tools.

Project surfaces

One storage format, two product faces

Java SDK

Integrate TinyLog into Java services with a business-facing logger API, config-driven appenders, masking, rotation, and direct .tog output.

  • YAML and properties config loading
  • Console and file backends
  • Level filtering and masking rules

Rust tools

Convert plaintext logs into .tog, restore them back when needed, and inspect them through a vim-like viewer optimized for large files.

  • tinylog-converter for convert and reverse restore
  • tinylog-viewer for browse, search, filter, and jump
  • Release packaging for direct binary download

Modules

Clean boundaries across the repository

tinylog-core

Domain model, codecs, and reader/writer contracts.

tinylog-sdk

tinylog-rust-common

Shared format parsing and protobuf support.

tinylog-converter

CLI for plaintext-to-`.tog` and reverse restore workflows.

tinylog-viewer

Interactive large-file viewer with trunk-ordered search.

Quick start

Try the current prototype fast

Convert a log

scripts/tinylog-convert.sh normal.log
scripts/tinylog-convert.sh --reverse normal.tog

Open a TinyLog file

scripts/tinylog-view.sh normal.tog
scripts/tinylog-open.sh normal.log

Build a release package

scripts/package-release.sh
